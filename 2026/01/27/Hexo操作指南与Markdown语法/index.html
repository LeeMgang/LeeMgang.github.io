第一部分：博客管理操作手册

这是我维护 Hexo 博客的标准流程（只维护 GitHub Pages 静态页面版）。

1. 新建文章

2. 写作与修改

直接使用 VS Code 打开 source/_posts 下对应的 .md 文件进行编辑。
写完后按 Ctrl + S 保存即可。

3. 删除文章

直接在文件管理器或 VS Code 左侧目录中，找到 source/_posts 下不需要的 .md 文件，右键删除即可。
注意：删除后需要重新生成并部署，线上才会消失。

第二部分：Markdown 语法速查

Markdown 是一种轻量级标记语言，让我们专注于写作。以下是常用语法演示。

1. 标题

使用 # 的数量表示标题等级：

一级标题 (通常由 Hexo 自动生成文章标题)

二级标题 (常用作大章节)

三级标题 (常用作小节)

2. 字体样式

  • 加粗：两边加两个星号 **内容**
  • 斜体：两边加一个星号 *内容*
  • 删除线：两边加两个波浪号 ~~内容~~

3. 列表

无序列表 (使用减号 - 加空格)：

  • 第一点
  • 第二点
    • 缩进两个空格可以嵌套

有序列表 (使用数字 1. 加空格)：

  1. 第一步
  2. 第二步
  3. 第三步

4. 引用

使用 > 符号：

这是一个引用段落。
知识就是力量。

5. 插入链接和图片

  • 插入链接：[显示文本](链接地址)
    例如：我的博客

  • 插入图片：![图片描述](图片链接)
    提示：如果不使用图床，可以将图片放在 source/images 文件夹下（需新建），然后引用 /images/图片名.jpg

6. 代码块

这是程序员最常用的功能。

行内代码：使用反引号 包裹。 例如：请执行git status` 命令。

代码块：使用三个反引号包裹，并指定语言。
